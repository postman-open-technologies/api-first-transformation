API QA Engineering 101

Shifting quality left across the the API lifecycle for all your APIs.

As enterprise organizations strive for higher levels of quality they are looking to get more structured and organized around how they are testing, collaborating more with engineers and other stakeholders, and shifting quality left across the API lifecycle across internal and external APIs.

  • Shifting quality left across the producer and consumer lifecycle
  • Hands-on: Integration Testing for APIs you consume (Consumer)
  • Hands-on: Functional Testing for APIs you produce (Producer)
  • Hands-on: Contract Testing for APIs you produce (Producer)
  • Hands-on: Testing Automation for APIs (Producer)
  • Testing requires a shared understanding of what quality means

Attendees should leave this portion of the training with clearer definitions of what API testing is, and possess some hands-on approaches they can be more collaborative with teams when it comes to quality and helping move testing earlier on in the API lifecycle, moving teams to be more proactive rather than reactive.

Learnings:

  • Apply integration testing using collections to the APIs you are consuming.
  • Apply functional testing using collections to the APIs you are producting.
  • Understand how API contracts can be used to standardize your testing.
  • Successfully automate your tests using runners, monitors, and CI/CD.
