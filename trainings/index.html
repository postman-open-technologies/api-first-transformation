Enterprise-grade trainings for engineers, QA, and product teams.

API Engineering 101

A baseline for all engineering teams in 2023 is that they have an awareness of the basics of consuming and producing APIs across a well-known API lifecycle, and segment of the trainings focus on what internal engineers will need to be successful in their work.

  • Strengthening the development across the API producer and consumer lifecycle
  • Hands-on: Building an app using a public API (Consumer)
  • Hands-on: Documenting, testing, and monitoring an existing API (Producer)
  • Hands-on: Documenting, testing, and monitoring an existing API using a proxy (Producer)
  • Hands-on: Prototyping a new API with Postman Collections (Producer)
  • Hands-on: Designing a new API with the OpenAPI Specification (Producer)
  • Produce or consume APIs with confidence in a collaborative way.

Walking away from this training any engineer should be confident in working with onboarding, troubleshooting, and consuming APIs, as well documenting, testing, and monitoring an existing API, and prototyping and designing entirely new APIs from scratch.

API QA Engineering 101

As enterprise organizations strive for higher levels of quality they are looking to get more structured and organized around how they are testing, collaborating more with engineers and other stakeholders, and shifting quality left across the API lifecycle across internal and external APIs.

  • Shifting quality left across the producer and consumer lifecycle
  • Hands-on: Integration Testing for APIs you consume (Consumer)
  • Hands-on: Functional Testing for APIs you produce (Producer)
  • Hands-on: Contract Testing for APIs you produce (Producer)
  • Hands-on: Testing Automation for APIs (Producer)
  • Testing requires a shared understanding of what quality means

Attendees should leave this portion of the training with clearer definitions of what API testing is, and possess some hands-on approaches they can be more collaborative with teams when it comes to quality and helping move testing earlier on in the API lifecycle, moving teams to be more proactive rather than reactive.

API Product Management 101

As part of their overall digital transformation enterprise organizations are find new velocity as well as revenue streams from treating externally facing APIs more as products, identifying more clearly who their consumers are, while establishing quantitative and qualitative feedback loops that they can use to more rapidly deliver high quality API resources.

  • Introducing API-as-a-Product
  • Hands-on: Publishing your first API (Producer view)
  • Hands-on: Distributing your first collection (Producer view)
  • Hands-on: Walking in customer shoes (Consumer view)
  • Hands-on: Iterating based on customer feedback (Producer view)
  • Evaluating quantitative and qualitative data for incremental development

This segment of the training should leave business and technical stakeholders with a more robust toolbox when it comes to publishing APIs publicly, being more thought with distribution, increasing their investment in developer experience, and improving the quality of quantitative and qualitative feedback used to iterate upon API resources.
