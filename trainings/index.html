The API-First Transformation Trainings

Enterprise-grade trainings for engineers, QA, and product teams.

API Engineering 101

A baseline for all engineering teams in 2023 is that they have an awareness of the basics of consuming and producing APIs, focusing on what internal engineers will need to be successful in their work.

  • Working with APIs is essential to your success as an engineer
  • Hands-on: Building an application using a simple public API
  • Hands-on: Documenting, testing, and monitoring an existing API
  • Hands-on: Design, document, and mock a new API for your team
  • Producing or consuming an API a collaborative way.

Walking away from this training any engineer should be confident in working with onboarding, troubleshooting, and consuming APIs, as well documenting, testing, and monitoring an existing API, and designing entirely new APIs from scratch.

API QA Engineering 101

As enterprise organizations strive for higher levels of quality they are looking to get more structured and organized around how they are testing, collaborating more with engineers and other stakeholders, and improving productivity across teams.

  • The impact of API quality on our businesses
  • Hands-on: Testing a single API that you depend on
  • Hands-on: Testing a single API you have created
  • Hands-on: Automating your work to scale its impact
  • Raising the overall quality bar in your API work

Attendees should leave this portion of the training with clearer definitions of what API testing is, and possess some hands-on approaches they can be more collaborative with teams when it comes to quality and helping move testing earlier on in the API lifecycle, moving teams to be more proactive rather than reactive.

API Product Management 101

As part of their overall digital transformation enterprise organizations are find new velocity as well as revenue streams from treating externally facing APIs more as products, identifying more clearly who their consumers are, while establishing quantitative and qualitative feedback loops that they can use to more rapidly deliver high quality API resources.

  • Introducing API-as-a-Product
  • Hands-on: Publishing your first API
  • Hands-on: Distributing your first collection
  • Hands-on: Walking in customer shoes
  • Hands-on: Iterating based on customer feedback
  • Evaluating quantitative and qualitative data for incremental development

This segment of the training should leave business and technical stakeholders with a more robust toolbox when it comes to publishing APIs publicly, being more thought with distribution, increasing their investment in developer experience, and improving the quality of quantitative and qualitative feedback used to iterate upon API resources.

API Architect 101

There are many ways that architects are contributing to API operations at scale across teams, but there are some essential ways enterprise organizations are starting building their API governance programs. API governance begins with standardizing how APIs are being built, helping enable teams as they are desiging, developing, and delivering new API features.

  • Getting started with API governance across the etnerprise
  • Hands-on: Defining machine-readable API contracts
  • Hands-on: Shaping API design and security using rules
  • Hands-on: Reporting on governance at enterprise scale
  • Enabling teams to do the right thing using common API standards.

This segment of the training should leave enterprise architects with a more robust toolbox when it comes to defining rules for how APIs should be delivered, enabling developers to do the right thing at design time, but also potentially enforcing at build time. Architects should walk away from this course with some actionable approaches to jumpstarting API governance at scale.
