API Engineering 101

Strengthening development across the producer and consumer lifecycle.

A baseline for all engineering teams in 2023 is that they have an awareness of the basics of consuming and producing APIs across a well-known API lifecycle, and segment of the trainings focus on what internal engineers will need to be successful in their work.

  • Strengthening the development across the API producer and consumer lifecycle
  • Hands-on: Building an app using a public API (Consumer)
  • Hands-on: Documenting, testing, and monitoring an existing API (Producer)
  • Hands-on: Documenting, testing, and monitoring an existing API using a proxy (Producer)
  • Hands-on: Prototyping a new API with Postman Collections (Producer)
  • Hands-on: Designing a new API with the OpenAPI Specification (Producer)
  • Produce or consume APIs with confidence in a collaborative way.

Walking away from this training any engineer should be confident in working with onboarding, troubleshooting, and consuming APIs, as well documenting, testing, and monitoring an existing API, and prototyping and designing entirely new APIs from scratch.

Learnings:

  • Establish an elevative view of being both an API producer and consumer.
  • Explore the friction that you have experienced as an API consumer.
  • Utilize your experience as an API consumer to develop empathy for consumers.
  • Understand there are multiple ways to producing an API with a common lifecycle.
